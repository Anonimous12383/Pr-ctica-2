# Práctica 2: Proyecto en Python y Configuración de Entorno

Descripción:
Cada estudiante debe configurar un entorno de desarrollo en Python y desarrollar un pequeño proyecto aplicando buenas prácticas de desarrollo. El proyecto debe incluir la gestión de dependencias, documentación adecuada y control de versiones con Git y GitHub. Finalmente, el repositorio del proyecto deberá ser subido a GitHub y compartido como evidencia.

Especificaciones:
Configurar un entorno virtual en Python y gestionar dependencias mediante la instalación de al menos una librería externa, esto tambien debe de incluir los archivos necesarios para la instalación posterior de dichas dependencias.

Incluir un archivo README.md la versión de python utilizada en el proyecto,  la descripción del proyecto, instrucciones de instalación y ejecución.

Agregar un archivo .gitignore para excluir archivos innecesarios.

Utilizar Git para realizar seguimiento del proyecto, asegurando un historial de cambios organizado.

Conectar el proyecto con un repositorio en GitHub y subir el código.

Compartir el enlace del repositorio en GitHub como evidencia de la entrega.

Criterios de Evaluación:
Uso correcto del entorno virtual y gestión de dependencias.

Documentación clara y estructurada en README.md.

Implementación adecuada del .gitignore.

Historial de commits organizado y significativo.

Repositorio en GitHub correctamente configurado y accesible.
